# NanumProject PRD

## 1) 목적
- 사내/프로젝트 단위 업무를 **Gantt 차트 기반**으로 관리하고, 프로젝트/태스크/멤버/권한을 한 곳에서 운영한다.
- 기존에 사용하던 **TeamGantt 서비스의 데이터를 Supabase로 마이그레이션**하여, 데이터 소유/확장성/자동화를 확보한다.

## 2) 대상 사용자
- 프로젝트 관리자(PM)
- 팀 멤버(업무 수행자)
- 관리자(Admin: 테넌트/멤버/권한/설정 관리)

## 3) 핵심 기능 (MVP)
### 프로젝트
- 프로젝트 목록/검색/필터
- 프로젝트 상세(기본 정보, 기간, 상태)
- 프로젝트 멤버 관리(역할, 활성/비활성)

### Gantt / 태스크
- Gantt 기반 태스크 시각화
- 태스크 CRUD(제목, 기간, 담당자, 진행률, 의존성)
- 진행률/상태 집계(대시보드 통계)

### 인증/권한
- Supabase Auth 기반 로그인/회원가입/비밀번호 재설정
- 멀티테넌트 지원(tenant_id 기반 데이터 격리)
- 역할 기반 권한(RBAC: admin/pm/user 등)

## 4) 비기능 요구사항
- **보안:** 모든 주요 테이블에 tenant_id 포함 + RLS 적용
- **감사/추적:** 주요 변경(프로젝트/태스크/멤버) 이벤트 로그(추후)
- **배포:** Vercel

## 5) 현재 상태(메모)
- 초기에는 Auth0-like IDaaS(NanumAuth/CoreAuth)를 구상했으나,
  TeamGantt 데이터 마이그레이션 이후 **NanumProject(프로젝트 관리)** 로 제품 방향이 확정됨.
- 인증 관련 문서는 `docs/auth/` 아래에 reference로 보관.
